## Configuration Overview

Vibe Coding offers extensive configuration options to customize your AI-assisted editing experience. Proper configuration ensures optimal performance, security, and integration with your preferred AI coding assistants.

<Steps>
  <Step title="Access Settings" icon="settings">
    Navigate to Vibe Coding > Settings in your WordPress admin dashboard.
  </Step>
  <Step title="AI Assistant Setup" icon="cpu">
    Configure your AI assistant connections and preferences.
    
    <Tabs>
      <Tab title="Cursor Configuration" icon="terminal">
        <ParamField body="apiKey" param-type="string" required="true">
          Your Cursor API key for authentication.
        </ParamField>
        <ParamField body="model" param-type="string" required="false">
          Preferred Cursor model (default: gpt-4).
        </ParamField>
        <ParamField body="timeout" param-type="integer" required="false">
          Request timeout in seconds (default: 30).
        </ParamField>
      </Tab>
      <Tab title="Claude Code Setup" icon="code">
        <ParamField body="apiKey" param-type="string" required="true">
          Your Claude API key.
        </ParamField>
        <ParamField body="maxTokens" param-type="integer" required="false">
          Maximum tokens per request (default: 4096).
        </ParamField>
        <ParamField body="temperature" param-type="number" required="false">
          Response creativity level (0.0-1.0, default: 0.7).
        </ParamField>
      </Tab>
    </Tabs>
  </Step>
  <Step title="Security Settings" icon="shield">
    Configure security parameters and access controls.
    
    <Expandable title="File Permissions" default-open="false">
      <ParamField body="allowedExtensions" param-type="array" required="false">
        File extensions allowed for editing (default: php, css, js, html).
      </ParamField>
      <ParamField body="restrictedPaths" param-type="array" required="false">
        Paths that cannot be edited (e.g., wp-config.php).
      </ParamField>
    </Expandable>
  </Step>
  <Step title="Backup Configuration" icon="save">
    Set up automatic backups and recovery options.
    
    <ParamField body="autoBackup" param-type="boolean" required="false">
      Enable automatic backups before changes (default: true).
    </ParamField>
    <ParamField body="backupRetention" param-type="integer" required="false">
      Number of backup versions to retain (default: 10).
    </ParamField>
  </Step>
</Steps>

## Environment-Specific Configurations

<Tabs>
  <Tab title="Development" icon="code">
    ```json
{
  "environment": "development",
  "debugMode": true,
  "autoBackup": false,
  "allowedExtensions": ["php", "css", "js", "html", "txt"]
}
    ```
  </Tab>
  <Tab title="Staging" icon="server">
    ```json
{
  "environment": "staging",
  "debugMode": false,
  "autoBackup": true,
  "backupRetention": 20
}
    ```
  </Tab>
  <Tab title="Production" icon="globe">
    ```json
{
  "environment": "production",
  "debugMode": false,
  "autoBackup": true,
  "restrictedPaths": ["wp-config.php", "wp-admin"]
}
    ```
  </Tab>
</Tabs>

## Troubleshooting Configuration Issues

<ExpandableGroup>
  <Expandable title="Connection Problems" default-open="false">
    Verify API keys and network connectivity.
    
    ```bash
# Test API connection
curl -X POST https://api.vibe-coding.com/test \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"test": "connection"}'
    ```
  </Expandable>
  <Expandable title="Permission Errors" default-open="false">
    Check WordPress file permissions and user capabilities.
    
    ```bash
# Check file permissions
ls -la wp-content/plugins/vibe-coding/
    ```
  </Expandable>
</ExpandableGroup>

<Callout kind="info">Configuration changes take effect immediately, but always test in a staging environment before applying to production.</Callout>

## Configuration Validation

Vibe Coding includes built-in validation to ensure your settings are optimal and secure. The plugin will alert you to potential issues and suggest improvements.

<Callout kind="tip">Use the configuration validator in Settings > Validation to check your current setup.</Callout>
